# Personal Life OS — Dashboard 設計審查

**視角：** Senior Product Designer  
**目標：** 強烈每日焦點感、清晰視覺層級、簡潔有力、Mobile-first

---

## 現況摘要

Dashboard 以垂直堆疊顯示三個等重區塊：Top 3、Time Log 摘要、Quick Links。全部使用相同卡片樣式。無日期脈絡、無進度指示、無快速操作。版面感覺像一般設定頁，而非每日指揮中心。

---

## 1. 版面改進

### 1.1 Hero 區：「今天」的識別

**問題：** 沒有「今天」的感覺。頁面可能是任何一天。使用者需要立即定位。

**解法：** 頂部加一個緊湊的日期 hero——mobile-first、拇指友善。

```
┌─────────────────────────────────────┐
│  星期六 · 2 月 28 日                 │  ← 單行、大點擊區
│  下午好                              │  ← 時段問候（可選）
└─────────────────────────────────────┘
```

- 使用 `toLocaleDateString('zh-TW', { weekday: 'long', month: 'long', day: 'numeric' })`
- 問候：早上（12 前）、下午（12–18）、晚上（18+）
- 不加多餘裝飾——只靠字型層級：日期主、問候次

### 1.2 手機 Z 型閱讀順序（優先順序）

**現況：** Top 3 → Time Log → Quick Links。視覺權重相同。

**建議順序（上到下）：**

1. **Date hero**（定位）
2. **Top 3**（每日焦點——「為什麼打開 app」）
3. **Focus metric**（Time Log——「做得如何」）
4. **Quick Links**（工具——次要，可收合或移到 below fold）

理由：使用者打開 app 想看「今天重要的事」和「有沒有在軌道上」。Quick Links 是輔助，非主體。

### 1.3 桌面網格（648px+）

**現況：** 所有 breakpoint 單欄。浪費橫向空間。

**建議：**

```
┌──────────────────────┬──────────────────────┐
│  Today · 2/28        │  Focus: 2h 15m       │  ← 日期 + 指標並排
├──────────────────────┴──────────────────────┤
│  Top 3 (全寬、突出)                          │
├──────────────────────┬──────────────────────┤
│  Quick Links         │  [開始計時] CTA       │  ← 行動導向
└──────────────────────┴──────────────────────┘
```

- 頂列：日期左、focus 指標右
- Top 3：全寬、最大卡片
- 底列：Quick Links + 突出的「開始計時」或「記錄時間」CTA

### 1.4 Safe Area 與底部導覽留白

**現況：** `pb-20` 給底部導覽。在瀏海機上內容可能顯得擠。

**改進：** 使用 `env(safe-area-inset-bottom)`，確保最後區塊有 `padding-bottom: max(1.5rem, env(safe-area-inset-bottom))`，避免內容被導覽或 home indicator 遮住。

## 2. 元件重設計

### 2.1 Top 3 區塊

**現況：** 編號列表配小圓圈。完成項目有刪除線。像一般清單。

**重設計：**

- **空狀態：** 大而友善的 CTA：「設定今日 Top 3」配箭頭或圖示。連結到 /tasks。加一行價值主張：「每天只做三件最重要的事」
- **有資料時：**
  - 每項：`[ 1 ]  任務標題`，左側 **checkbox**（點擊完成）
  - 使用 **進度指標**：「1/3 完成」或細進度條（33%、66%、100%）
  - 第一個未完成項：略粗或軟 highlight（如 `bg-indigo-500/5`）
  - 已完成：變暗 + 刪除線，但仍可見（成就感）

**視覺層級：**
- 區塊標籤：「今日 Top 3」— 小、大寫或 small-caps、tracking-wide、muted
- 項目：16–18px，舒適點擊區（最小 44px 高）
- 數字：24–28px、粗體、accent 色——「1、2、3」要有里程碑感

### 2.2 Time Log / Focus 指標

**現況：** 大數字 +「專注時間」+ 連結。與行動脫節。

**重設計：**

- **主體：** 大數字（如「2h 15m」）— 作為 hero 指標。若有每日目標可考慮圓形或半圓進度（如「4h 目標」）
- **次要：** 「今日專注時間」— 下方小標籤
- **行動：** 將「查看詳情 →」換成 **主要 CTA 按鈕**：「開始計時」或「記錄時間」
  - 若計時中：顯示「● 計時中 · 00:15:32」+「停止」按鈕
- **空狀態（0m）：** 「還沒開始記錄」+「開始第一筆」按鈕。避免罪惡感；用鼓勵文案。

### 2.3 Quick Links

**現況：** Flex-wrap 邊框 chips。權重相同。

**重設計：**

- **手機可收合：** 「快速連結」配 chevron。點擊展開。預設：若 > 4 個連結則收合，或顯示前 4 個 +「更多」
- **視覺：** 降低邊框存在感。圖示 + 標籤。可考慮 2 欄網格增加可預測性
- **層級：** 若有「最愛」，優先顯示。或允許使用者排序（未來）
- **空狀態：** 「新增常用連結」+ 連結到 /settings 的 + 圖示

### 2.4 卡片一致性

**現況：** 所有區塊用 `rounded-xl border border-zinc-800 bg-zinc-900/50`。處理相同。

**改進：** 依重要性區分：

- **Top 3：** 略突出—`shadow-md` 或 `ring-1 ring-indigo-500/10` 表示「這是主體」
- **Focus 指標：** 可更簡潔（少邊框），數字是焦點
- **Quick Links：** 最輕—可能只用 subtle divider，不用完整卡片

## 3. 視覺回饋構想

### 3.1 完成回饋

- **Top 3 checkbox 點擊：** 短暫勾選動畫（如 scale 0→1 ease-out）或 subtle confetti/particle（別過度）
- **聲音：** 完成時可選 haptic（`navigator.vibrate(50)`）— 尊重使用者偏好
- **進度：** 3/3 完成時顯示小「今日 Top 3 完成！」badge 或 micro-celebration

### 3.2 載入狀態

- **現況：** 「載入中...」文字。感覺突兀。
- **改進：** Skeleton placeholders。如 Top 3 三條灰條、指標用灰圓。保留版面、體感更快。

### 3.3 計時器回饋

- 計時中：Dashboard 上脈動點或 subtle 動畫，讓使用者一眼看到「有東西在跑」
- 可考慮計時中時頂部 compact timer bar（sticky、極簡）

### 3.4 空狀態

- **Top 3 空：** 插圖或圖示 +「設定今日 Top 3」+ 簡短好處
- **無連結：** 「新增常用連結，一鍵開啟」+ CTA
- **0 專注時間：** 「開始記錄，看見自己的專注」— 正向框架

### 3.5 微互動

- **連結 hover/tap：** 輕微 scale（1.02）或背景 transition
- **區塊展開/收合：** 平滑高度 transition（max-height + overflow）
- **數字變化：** 專注時間更新時可考慮 subtle 數字遞增動畫（可選，別過度）

## 4. 動機機制

### 4.1 每日進度

- **Top 3 進度：** 「1/3」或「2/3」— 可見完成比例
- **連續天數（未來）：** 「連續 3 天完成 Top 3」— 簡單 badge。在 persistence 存 `lastCompletedDate` 和 `streakCount`
- **專注目標（可選）：** 「今日目標 4h」配進度環。使用者在設定中設定。Dashboard 顯示「2h 15m / 4h」

### 4.2 正向強化

- **完成訊息：** 使用者完成 Top 3 項目時：「✓ 完成！」toast 或 inline 訊息。保持簡短
- **一天結束（未來）：** 若 3 個都完成：「今日 Top 3 全部完成 🎉」— 慶祝但不幼稚

### 4.3 低壓力設計

- 對「0 專注時間」或「無 Top 3」避免紅或警告色。用中性或鼓勵色調
- 「還沒開始」而非「未完成」或「失敗」
- 進度條用填滿（正向）而非倒數（壓力）

### 4.4 快速勝利

- **一鍵開始計時：** 從 dashboard「開始計時」可開小 modal：「做什麼？」+ 分類 + 開始。減少步驟
- **快速完成：** 允許在 dashboard 完成 Top 3，無需導航到 /tasks

---

## 5. 互動改進

### 5.1 點擊目標

- 所有互動元素（checkbox、連結、按鈕）**最小 44×44px**
- Top 3 項目：整列可點，不只 checkbox
- Quick link chips：足夠 padding（已有 px-3 py-2；確保觸控區足夠）

### 5.2 滑動 / 手勢（可選）

- **右滑完成：** Top 3 項目右滑標記完成（像 todo app）
- **左滑刪除：** Quick Link 左滑刪除（帶 undo toast）
- 用 `touchstart` / `touchmove` / `touchend` 或輕量套件實作。不阻擋核心流程。

### 5.3 下拉更新

- 手機上下拉更新 dashboard 資料（reload hooks）。標準模式；減少「是不是過期了？」的焦慮。

### 5.4 鍵盤 / 無障礙

- 確保 focus 順序：日期 → Top 3（1、2、3）→ Focus 指標 → CTA → Quick Links
- 僅圖示按鈕加 `aria-label`
- Top 3：`role="list"` 和 `aria-label="今日 Top 3"`

### 5.5 Deep Links

- 「開始計時」可 deep link 到 /log 帶 `?action=start` 自動開啟計時 modal
- 「設定 Top 3」連結到 /tasks 帶 `?filter=top3` 只顯示 Top 3 任務

## 實作優先順序

| 優先 | 項目 | 工時 | 影響 |
|------|------|------|------|
| P0 | Date hero + 問候 | 低 | 高（定位） |
| P0 | Top 3 進度（1/3、2/3、3/3） | 低 | 高（焦點） |
| P0 | 「開始計時」CTA 取代「查看詳情」 | 低 | 高（行動） |
| P1 | Top 3 checkbox / dashboard 上點擊完成 | 中 | 高 |
| P1 | Skeleton loading | 低 | 中 |
| P1 | 區分卡片層級 | 低 | 中 |
| P2 | Quick Links 可收合 | 低 | 中 |
| P2 | 空狀態文案 + CTA | 低 | 中 |
| P3 | 專注目標 + 進度環 | 中 | 中 |
| P3 | 連續天數 badge | 中 | 低（動機） |
| P3 | 下拉更新 | 中 | 低 |

---

## 總結

Dashboard 應像 **「今天的指揮中心」**——立即定位（日期）、清晰焦點（帶進度的 Top 3）、直接行動路徑（開始計時）。減少視覺噪音、突出最重要元素、加入輕量動機機制但不雜亂。Mobile-first 意味：大點擊區、可收合次要內容、直式單手可操作的版面。
